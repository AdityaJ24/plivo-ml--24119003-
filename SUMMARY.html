Error Analysis & Targeted Feature Iterations (Run 5)

To break through performance plateaus, we conducted systematic out-of-fold error analysis on misclassified pauses:

Discovered Failure Modes & Targeted Engineering Solutions

  • False Positives (Hold Pauses misclassified as EOT): Occurred on mid-sentence holds where speech ended on low or unvoiced pitch.
    Fix: Formulated st_last (speaker-normalized semitone pitch $ST = 12 \log_2(F_0 / F_{0, mean\_30})$) and e_tail_silence_ms (trailing quiet frames $< -45$dB immediately before pause onset). True EOT endings exhibit $50-150$ms smooth trailing silence decay.
  • False Negatives (EOT turns misclassified as Hold): Occurred on question turns with rising pitch intonation ($ST_{rise} > 3$ semitones) where terminal pitch did not drop.
    Fix: Formulated st_rise_indicator to explicitly capture question intonation, paired with short-term energy drop features (e_drop_100_300, e_drop_100_1500).
  • Hesitation Gaps & Timbral Relaxation: Mid-turn holds often follow preceding micro-pauses ("uh", "um"). We added unvoiced_gaps_15 / unvoiced_gaps_30 and $\Delta\text{MFCC}$ timbral acceleration features.

Library Compliance & Causality Audit

✓ Full Library Compliance Achieved

All code uses only allowed libraries: numpy, scipy, scikit-learn, joblib. No LightGBM, CatBoost, Whisper, wav2vec, Silero, or pretrained models are used. The model artifact (eot_model.joblib) contains only scikit-learn estimators (ExtraTreesClassifier, GradientBoostingClassifier, HistGradientBoostingClassifier, MLPClassifier, RandomForestClassifier, LogisticRegression) wrapped in a soft VotingClassifier.

Hard Causality Enforcement (t ≤ pause_start)

Every feature calculation in features_advanced.py adheres to strict causality:

  1. Strict Sample Slicing: audio_full = x[:int(pause_start * sr)] and get_speech_before() slice x[start:end] where end = int(pause_start * sr). Python's slice excludes index end, ensuring zero audio samples at or beyond pause_start are ever read.
  2. Zero Reference to Future Metadata: pause_end and pause duration are never referenced in feature extraction or prediction code.
